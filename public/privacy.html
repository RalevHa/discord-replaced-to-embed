Privacy Policy

What this bot looks at, what it keeps, and for how long.

In-memory by default; persists only if the operator enables Upstash

/01

What we process

  • Message content — scanned in real time to detect supported links (TikTok, X, Bilibili, Pixiv, Bluesky, Facebook) and, if spam detection is enabled, to spot the same text posted across many channels in a short window. Message text is not stored after processing; it's only kept in memory briefly to make that decision.
  • Server, channel, and user IDs — used to know where to reply, which server has disabled auto-conversion, and who to time out if flooding is detected. Not linked to any identity beyond the Discord IDs Discord itself already assigns.
  • Scraped Facebook page data — when converting a Facebook link, the Bot fetches that public page's Open Graph tags (title, description, image/video URL) to build an embed. This is public data from the linked page itself, not from your Discord account.

/02

What we store

By default, all state (per-server on/off toggle, conversion counters, flood-detection history) lives in memory only and is lost on restart. If the Bot's operator configures an optional Upstash Redis database, the following persist across restarts instead:

  • Per-server enabled/disabled state (server ID only).
  • Aggregate conversion counts (numbers only, no message content).

No message content, attachments, or usernames are ever written to persistent storage.

/03

Who can see it

Only the Bot's operator has access to the Upstash database (if configured) and server console logs. Moderation alerts (for spam-flooding incidents) are posted only to the mod-log channel a server admin configures — visible to that server's own moderators, not to the Bot's operator.

/04

Third parties

The Bot fetches pages from the platforms it supports (e.g. facebook.com) and relies on independent, community-run "fixup" services (e.g. fixupx.com, vxbilibili.com) to render embeds for other platforms. Those services see the URL you shared, per their own privacy practices — the Bot does not send them any Discord account information.

/05

Data retention & deletion

Removing the Bot from a server stops all processing for that server immediately. If persistence is enabled, ask the Bot's operator to delete that server's stored toggle state/counters, or they're cleared naturally by disabling persistence.

/06

Children's privacy

The Bot doesn't knowingly collect data from anyone under Discord's own minimum age requirement (13, or higher where required by local law).

/07

Changes

This policy may be updated as the Bot's features change. Continued use after an update means you accept the revised policy.

/08

Contact

Questions or data-deletion requests: open an issue on the project's GitHub repository.

อยู่ในหน่วยความจำเท่านั้น เว้นแต่ผู้ดูแลเปิดใช้ Upstash

/01

ข้อมูลที่เราประมวลผล

  • เนื้อหาข้อความ — ถูกตรวจสอบแบบเรียลไทม์เพื่อค้นหาลิงก์จากแพลตฟอร์ม ที่รองรับ (TikTok, X, Bilibili, Pixiv, Bluesky, Facebook) และหากเปิดใช้งานระบบตรวจจับ สแปม จะใช้ตรวจสอบว่ามีการโพสต์ข้อความเดียวกันในหลายช่องทางในช่วงเวลาสั้น ๆ หรือไม่ ข้อความจะไม่ถูกจัดเก็บหลังการประมวลผล เก็บไว้ในหน่วยความจำชั่วคราวเพื่อใช้ตัดสินใจเท่านั้น
  • ID ของเซิร์ฟเวอร์ ช่อง และผู้ใช้ — ใช้เพื่อทราบว่าจะตอบกลับที่ไหน เซิร์ฟเวอร์ใดปิดการแปลงลิงก์อัตโนมัติไว้ และจะ timeout ผู้ใช้คนใดหากตรวจพบการสแปม ไม่มีการเชื่อมโยงกับตัวตนใด ๆ นอกเหนือจาก ID ที่ Discord กำหนดไว้อยู่แล้ว
  • ข้อมูลหน้า Facebook ที่ดึงมา — เมื่อแปลงลิงก์ Facebook บอทจะดึงแท็ก Open Graph ของหน้าสาธารณะนั้น (ชื่อเรื่อง คำอธิบาย ลิงก์รูปภาพ/วิดีโอ) เพื่อสร้าง embed ซึ่งเป็นข้อมูลสาธารณะจากหน้าเพจนั้นเอง ไม่ใช่จากบัญชี Discord ของคุณ

/02

ข้อมูลที่เราจัดเก็บ

โดยค่าเริ่มต้น ข้อมูลสถานะทั้งหมด (การเปิด/ปิดการทำงานต่อเซิร์ฟเวอร์ ตัวนับการแปลงลิงก์ ประวัติการตรวจจับสแปม) จะถูกเก็บไว้ในหน่วยความจำเท่านั้นและหายไปเมื่อรีสตาร์ท หากผู้ดูแลบอทตั้งค่าฐานข้อมูล Upstash Redis เพิ่มเติม (ทางเลือก) ข้อมูลต่อไปนี้จะถูกเก็บไว้ข้ามการรีสตาร์ทแทน:

  • สถานะเปิด/ปิดของแต่ละเซิร์ฟเวอร์ (เฉพาะ ID เซิร์ฟเวอร์)
  • จำนวนการแปลงลิงก์สะสม (ตัวเลขเท่านั้น ไม่มีเนื้อหาข้อความ)

เราจะไม่บันทึกเนื้อหาข้อความ ไฟล์แนบ หรือชื่อผู้ใช้ลงในระบบจัดเก็บถาวรเด็ดขาด

/03

ใครสามารถเข้าถึงข้อมูลได้บ้าง

มีเพียงผู้ดูแลบอทเท่านั้นที่เข้าถึงฐานข้อมูล Upstash (ถ้ามีการตั้งค่า) และ log ของ console การแจ้งเตือน moderation (กรณีตรวจพบการสแปม) จะถูกโพสต์เฉพาะในช่อง mod-log ที่ผู้ดูแลเซิร์ฟเวอร์ตั้งค่าไว้เท่านั้น ซึ่งมองเห็นได้เฉพาะผู้ดูแลของเซิร์ฟเวอร์นั้น ไม่ใช่ผู้ดูแลบอท

/04

บุคคลที่สาม

บอทจะดึงข้อมูลจากแพลตฟอร์มที่รองรับ (เช่น facebook.com) และใช้บริการ "fixup" ที่ดำเนินการโดยชุมชนอิสระ (เช่น fixupx.com, vxbilibili.com) เพื่อสร้าง embed สำหรับแพลตฟอร์มอื่น ๆ บริการเหล่านั้นจะเห็นเฉพาะ URL ที่คุณแชร์ ตามนโยบายความเป็นส่วนตัว ของแต่ละบริการเอง โดยบอทจะไม่ส่งข้อมูลบัญชี Discord ใด ๆ ให้บริการเหล่านั้น

/05

การเก็บรักษาและการลบข้อมูล

การนำบอทออกจากเซิร์ฟเวอร์จะหยุดการประมวลผลทั้งหมดในเซิร์ฟเวอร์นั้นทันที หากเปิดใช้งานการจัดเก็บถาวร สามารถขอให้ผู้ดูแลบอทลบสถานะ/ตัวนับที่จัดเก็บไว้ของเซิร์ฟเวอร์ นั้นได้ หรือข้อมูลจะหายไปเองเมื่อปิดการใช้งานระบบจัดเก็บถาวร

/06

ความเป็นส่วนตัวของเด็ก

บอทไม่มีเจตนาเก็บข้อมูลจากผู้ที่มีอายุต่ำกว่าเกณฑ์ขั้นต่ำของ Discord (13 ปี หรือสูงกว่าตามที่กฎหมายท้องถิ่นกำหนด)

/07

การเปลี่ยนแปลง

นโยบายนี้อาจได้รับการปรับปรุงตามการเปลี่ยนแปลงของฟีเจอร์บอท การใช้งานต่อหลังจากมีการอัปเดตถือว่าคุณยอมรับนโยบายฉบับใหม่

/08

ติดต่อเรา

หากมีคำถามหรือต้องการขอให้ลบข้อมูล สามารถเปิด issue ได้ที่ GitHub repository ของโปรเจกต์
